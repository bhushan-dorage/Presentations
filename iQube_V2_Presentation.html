Audit Logging
Append-only, tamper-evident distributed audit trail — all service state changes flow through Kafka into a dedicated Audit Service

✅
Key Design Decisions
• Event-driven via Kafka — audit writing never blocks business ops
• Append-only — no UPDATE/DELETE ever runs on audit_log table
• SHA-256 checksum per row detects tampering post-insert
• Monthly partitions — keeps query performance stable as data grows
• @AuditAction AOP — annotate any service method for zero-boilerplate before/after capture
AuditEventPublisher.javaJava
@Component
public class AuditEventPublisher {

    @Autowired
    private KafkaTemplate<String,AuditEvent> kafka;

    public void publish(
            String entityType, UUID entityId,
            String action, Object before,
            Object after, String actorId,
            String correlationId) {

        var event = AuditEvent.builder()
            .auditId(UUID.randomUUID())
            .tenantId(TenantContext.get())
            .entityType(entityType)
            .entityId(entityId)
            .action(action)
            .actorId(actorId)
            .oldValue(toJson(before))
            .newValue(toJson(after))
            .correlationId(correlationId)
            .checksum(sha256(entityId, action, after))
            .eventTime(Instant.now())
            .build();

        // entityId as key → ordered per entity
        kafka.send("{tenant}.audit.event",
            entityId.toString(), event);
    }
}
AuditConsumer.javaJava
@Service
public class AuditConsumer {

    @KafkaListener(topics="#{topics.audit}",
                   groupId="audit-svc-group")
    public void consume(AuditEvent event) {
        // 1. Idempotency — skip if already persisted
        if (repo.existsById(event.getAuditId())) return;

        // 2. Verify tamper-detection checksum
        if (!verifyChecksum(event))
            throw new AuditIntegrityException(event);

        // 3. Insert — never UPDATE or DELETE
        repo.save(toEntity(event));
    }
}

Redis Config Caching
Eliminate repeated DB reads for static config — serve at sub-millisecond latency with event-driven cache invalidation

⚡
Cache-Aside Pattern
HIT → return from Redis (<1ms). MISS → load from DB → store in Redis with TTL → return. Config change → service publishes cache.invalidation Kafka event → all service instances evict matching keys. TTL is the safety-net fallback.
🏗️
Redis Cluster Topology
3 masters + 3 replicas. ~5460 hash slots per master. Automatic failover on master loss. AOF + RDB persistence for durability. Services fall back to DB if Redis is unavailable.
ConfigCacheService.javaJava
@Service
public class ConfigCacheService {

    @Autowired private RedisTemplate<String,Object> redis;
    @Autowired private IngestionConfigRepo configRepo;
    @Autowired private DataPermissionRepo  permRepo;

    private static final Duration TTL5 = Duration.ofMinutes(5);
    private static final Duration TTL10 = Duration.ofMinutes(10);

    /** Cache-aside: ingestion config */
    public IngestionConfig getIngestionConfig(
            String tenant, String configId) {

        var key = "config:ingest:%s:%s"
            .formatted(tenant, configId);

        // 1. Check Redis first
        var hit = (IngestionConfig)
            redis.opsForValue().get(key);
        if (hit != null) return hit;

        // 2. MISS: load from DB + populate cache
        var cfg = configRepo
            .findByTenantAndId(tenant, configId)
            .orElseThrow(ConfigNotFoundException::new);
        redis.opsForValue().set(key, cfg, TTL5);
        return cfg;
    }

    /** Cache-aside: role data permissions */
    public DataPermissions getPermissions(
            String tenant, String role) {

        var key = "config:perms:%s:%s"
            .formatted(tenant, role);
        var hit = (DataPermissions)
            redis.opsForValue().get(key);
        if (hit != null) return hit;
        var result = DataPermissions.from(
            permRepo.findAllByTenantAndRole(tenant,role));
        redis.opsForValue().set(key, result, TTL5);
        return result;
    }

    /** Kafka listener: invalidate on config change */
    @KafkaListener(topics="cache.invalidation")
    public void onInvalidation(
            CacheInvalidationEvent evt) {

        var pattern = "config:%s:%s:*"
            .formatted(evt.getType(), evt.getTenant());
        Set<String> keys = redis.keys(pattern);
        if (keys != null && !keys.isEmpty()) {
            redis.delete(keys);
            log.info("Evicted {} keys: {}",
                keys.size(), pattern);
        }
    }
}
Improvement #3 — Hardened
Data View SQL Injection Hardening
Column allowlist + JPA Criteria API + mandatory tenant predicate = zero SQL injection surface area
🚨
The Vulnerability — Config-Driven Dynamic SQL
The Data View Service builds queries dynamically from user-supplied column names and filter values. Concatenating these directly into SQL strings allows an attacker to inject arbitrary SQL — extracting other tenants' data, dropping tables, or bypassing RBAC entirely. Every column name must be validated against a strict allowlist. Every filter value must be a bound PreparedStatement parameter, never a string literal.
❌

SECURE — Allowlist + JPA Criteria API
SecureQueryBuilder.java✅ SAFE
@Component
public class SecureQueryBuilder {

    @Autowired private ConfigCacheService cache;
    @Autowired private EntityManager em;

    public Page<CaseView> query(
            DataViewRequest req,
            String tenant, String role) {

        // 1. Allowlist from Redis (fast cache)
        Set<String> allowed =
            cache.getPermissions(tenant, role)
                 .getVisibleColumns();

        // 2. Validate sort column
        if (req.getSortCol() != null
            && !allowed.contains(req.getSortCol()))
            throw new InvalidColumnException(req.getSortCol());

        // 3. Validate every filter column
        req.getFilters().keySet().forEach(col -> {
            if (!allowed.contains(col))
                throw new InvalidColumnException(col);
        });

        // 4. JPA Criteria API — safe PreparedStatement
        var cb   = em.getCriteriaBuilder();
        var q    = cb.createQuery(CaseMaster.class);
        var root = q.from(CaseMaster.class);
        List<Predicate> p = new ArrayList<>();

        // 5. Tenant isolation — ALWAYS applied
        p.add(cb.equal(root.get("tenantId"), tenant));

        // 6. Filters: bound parameters — NOT literals
        req.getFilters().forEach((col, val) -> {
            var field = cache
                .getIngestionConfig(tenant, col)
                .getEntityFieldName(); // from config, not user
            p.add(cb.equal(root.get(field), val));
        });

        q.where(p.toArray(new Predicate[0]));

        // 7. Sort field from config — not user input
        if (req.getSortCol() != null) {
            var f = cache
                .getIngestionConfig(tenant, req.getSortCol())
                .getEntityFieldName();
            q.orderBy(req.isAsc()
                ? cb.asc(root.get(f))
                : cb.desc(root.get(f)));
        }
        return paginate(q, req.getPageable());
    }
}
🛡️
Column Allowlist
Only columns defined in UI config are accepted. Any other column name in the request → 400 Bad Request before the SQL layer is ever reached.
🔒
Parameterized Values
JPA Criteria API generates PreparedStatements. Filter values are bound parameters — never interpolated into SQL strings under any execution path.
🏠
Mandatory Tenant Isolation
tenantId predicate is always applied and cannot be overridden by any request parameter. Cross-tenant data leakage is architecturally impossible.

Circuit Breaker — Resilience4J
All synchronous inter-service calls wrapped. OPEN state serves cached fallback response. Prevents cascading failure when a downstream service degrades.
🔁
Kafka DLQ + Retry
3 retries with exponential backoff (1s→2s→4s). After max retries, message routes to {tenant}.dlq.*. Ops replays after root-cause fix.
🗄️
PostgreSQL HA — Patroni + HAProxy
1 Primary + 2 Synchronous Replicas. Patroni auto-promotes on primary failure (~30s). HAProxy routes writes to primary, reads to replicas.
📦
Pod Anti-Affinity + PDB
PodAntiAffinity spreads replicas across nodes. PodDisruptionBudget minAvailable=2 prevents simultaneous eviction during K8s node maintenance.
💾
Redis Fallback
If Redis is unavailable, services fall back to DB queries for config — degraded performance but full functionality preserved. Prometheus alert fires immediately.
